工作中C# MVC接口路由的一些命名习惯

发表于 2019-08-15

1.接口路由名和方法名保持一致（[Route(“get_event_join_list”)]和get_event_join_list）要保持一致

[Route("get_event_join_list")]
[HttpGet]
public DataResponse<PageResponse<EventDJoinViewModel>> get_event_join_list([FromUri]EventJoinSearchViewModel search)
{
    var searchDTO = search.Map<EventJoinSearchViewModel, EventJoinSearchDTO>(Constant.MapType.LowerToPascal);
    var result = Business.Logic<EventLogic>().GetEventJoinList(searchDTO, AppKey);

    return new DataResponse<PageResponse<EventDJoinViewModel>>
    {
        success = true,
        code = ResponseCode.Ok.code,
        message = ResponseCode.Ok.message,
        data = new PageResponse<EventDJoinViewModel>
        {
            last_page = searchDTO.PageIndex * searchDTO.PageSize >= searchDTO.TotalCount,
            total_count = searchDTO.TotalCount,
            list = result.Map<EventDJoinDTO, EventDJoinViewModel>(Constant.MapType.PascalToLower)
        }
    };
}

1.1接口路由方法名也要和逻辑层和仓存储的方法保持一致，如上面的get_event_join_list和GetEventJoinList

逻辑层：
public List GetEventJoinList(EventJoinSearchDTO search, string appKey)
{
return new EventRepository().GetEventJoinList(search, appKey).ToList();
}
仓存储：
public IQueryable GetEventJoinList(EventJoinSearchDTO search, string appKey)
{
var query = from e in GetAll()
join ej in DBContext.EventJoins on e.EventId equals ej.EventId
join t in DBContext.EventTypes on e.EventTypeId equals t.EventTypeId
join tg in DBContext.EventTypeGroups on t.EventTypeGroupId equals tg.EventTypeGroupId
where e.AppKey == appKey && !e.DateDeleted.HasValue
orderby ej.OrderId
select new EventDJoinDTO
{
AppKey = e.AppKey,
Title = e.Title,
EventAddress = e.EventAddress,
Description = e.Description,
EventId = e.EventId,
ShopId = e.ShopId,
OrderId = ej.OrderId,
UserId = ej.UserId,
DateDeleted = e.DateDeleted,
DateCreated = e.DateCreated,
ClientId = e.ClientId,
CategoryId = e.CategoryId,
Price = e.Price,
EventTypeId = e.EventTypeId,
EventTypeName = t.EventTypeName,
EventTypeGroupId = tg.EventTypeGroupId,
EventTypeGroupName = tg.EventTypeGroupName,
UserName = ej.UserName,
Mobile = ej.Mobile,
Note = ej.Note,
ImagePaths = ej.ImagePaths,
VideoJson = ej.VideoJson,
EventDateFrom = e.EventDateFrom,
EventDateTo = e.EventDateTo
};

    if (search.EventId.HasValue)
    {
        query = query.Where(r => r.EventId == search.EventId);
    }
    if (search.OrderId.HasValue)
    {
        query = query.Where(r => r.OrderId == search.OrderId);
    }
    if (search.EventTypeId.HasValue)
    {
        query = query.Where(r => r.EventTypeId == search.EventTypeId.Value);
    }
    if (search.EventTypeGroupId.HasValue)
    {
        query = query.Where(r => r.EventTypeGroupId == search.EventTypeGroupId);
    }
    if (search.UserId.HasValue)
    {
        query = query.Where(r => r.UserId == search.UserId);
    }
    if (!string.IsNullOrWhiteSpace(search.Mobile))
    {
        query = query.Where(r => r.Mobile.Contains(search.Mobile));
    }

    if (search.PageIndex > 0)
    {
        search.TotalCount = query.Count();
        query = query.Skip((search.PageIndex - 1) * search.PageSize).Take(search.PageSize);
    }

    return query.AsNoTracking();
}

1.2接口(api的controller层)路由方法传入单个参数的时候要用（int order_id，int event_id）这样的命名方式

[Route("get_event_join_detail")]
[HttpGet]
public DataResponse<EventDJoinViewModel> get_event_join_detail(int order_id)
{
    var result = Business.Logic<EventLogic>().GetEventDJoinDetail(new EventJoinSearchDTO { OrderId = order_id }, AppKey);

    return new DataResponse<EventDJoinViewModel>
    {
        success = true,
        code = ResponseCode.Ok.code,
        message = ResponseCode.Ok.message,
        data = result.Map<EventDJoinDTO, EventDJoinViewModel>(Constant.MapType.PascalToLower)
    };
}

1.2.1后台controller层传入单个参数的时候可以用(int eventId)这样的方式，第一个字母小写，之后单词的第一个字母都大写

public ActionResult Forbidden(long eventId)
{
var dto = new ForbiddenEventViewModel { event_id = eventId, user_id = UserId.Value };
var obj = Business.Gateway().ForbiddenEvent(dto);
return Json(new BaseResponse { success = obj.success, code = obj.code, message = obj.message });
}

2.ViewModel和DTO自由属性的命名规范

2.1Viewmodel:(根据需要返回需要的属性，给Controller层传参或者返回)

using System;
namespace AlaGou.Admin.ViewModel
{
public class HouseViewModel
{
public int? house_id { get; set; }
public string title { get; set; }
public int client_id { get; set; }
public string price { get; set; }
public string tag { get; set; }
public DateTime date_created { get; set; }
}
public class HouseDetailViewModel : HouseViewModel
{
public string cover_image_path { get; set; }
public string description { get; set; }
}
}

2.2DTO:(和数据库里设计的属性保持一致，给逻辑层和仓储层用的对象)
///HouseDTO(如下)，可以直接继承House表（数据库里自己设计的表，一般在一个项目的Model中），省去再写属性的麻烦
using System.ComponentModel.DataAnnotations.Schema;
using AlaGou.Models;
namespace AlaGou.BusinessLogic.DTO
{
[NotMapped]
public class HouseDTO : House
{
}
}
///HouseEditDTO（如下）
using AlaGou.Models;
namespace AlaGou.BusinessLogic.DTO
{
public class HouseEditDTO : House
{
}
}
///HouseSearchDTO（如下）
using ITPlus.Contract;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
namespace AlaGou.BusinessLogic.DTO
{
public class HouseSearchDTO : MvcPaging
{
public string Title { get; set; }

}
}
